### Stress Test Findings

**auth (VITAL_20):** Genuine leverage — auth touches every request path. Not convention-driven; this is the real attack surface.

**billing (VITAL_20):** [CONVENTIONAL]: "Billing is complex, bugs are expected." [PARETO REFRAME]: 31 bugs in billing suggests the payment state machine has structural holes. A focused invariance audit of the state transitions would collapse bug rate faster than incremental patching.

**search (VITAL_20):** [CONVENTIONAL]: "Search bugs are edge cases from weird queries." [PARETO REFRAME]: 22 bugs in search likely trace to 2-3 missing input validation paths. The bug surface is narrower than it appears — most search bugs are variants of the same root cause.

**email-templates (TRIVIAL_80):** 9 bugs, but these are cosmetic/rendering. Done from habit of "fix every reported bug equally." Stopping entirely for one sprint costs nothing user-facing.

### Causal Leverage Map

**auth → permissions cascade:** Auth and permissions share the same session/token layer. 15 permissions bugs likely include 5-8 that are actually auth bugs misclassified by the triage team. Fixing auth's token validation would cascade-fix a subset of permissions bugs without touching the permissions module directly.

**billing → notifications dependency:** Billing state changes trigger notification events. 6-8 of the 18 notification bugs are downstream of billing state machine errors. Fix billing first → notification bug count drops without direct intervention.

**search → analytics feedback loop:** Search query logging feeds analytics. Malformed search queries that cause search bugs also corrupt analytics data. The 8 analytics bugs may partially resolve when search input validation is fixed.

### Unconventional Reframes

[CONVENTIONAL]: Distribute engineers proportionally across bug counts → [PARETO REFRAME]: Concentrate all 4 engineers on 3 modules. The cascade effects between auth→permissions and billing→notifications mean fixing 3 modules effectively addresses 5.

[CONVENTIONAL]: Triage every bug independently → [PARETO REFRAME]: 30-40% of bugs across notifications, permissions, and analytics are symptoms of root causes in auth, billing, and search. Bug triage is over-counting the true defect surface by ~35%.

[CONVENTIONAL]: A "stability sprint" means reducing total bug count → [PARETO REFRAME]: The metric that matters is bug severity × user impact, not count. Auth and billing bugs block revenue paths. The 38 bugs in TRIVIAL_80 modules combined have less user impact than auth's 47 alone.
